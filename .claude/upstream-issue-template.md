# Upstream issue template

Upstream CONTRIBUTING.md: "The best way to contribute is to open a Github issue. Bug reports,
feature requests, ideas are welcome." — this is the ONLY sanctioned upstreaming path.

Draft one issue per FORK_FEATURES.md slice or per bug. Keep it self-contained; never reference
our fork, branches, or PRs.

---

**Title:** [bug|idea] <one line, their voice: direct, technical>

**What happens / what's missing:**
<2-4 sentences. For bugs: exact repro (project fps, clip setup, tool call or UI steps) + observed
vs expected. For ideas: the user problem, not our implementation.>

**Why it matters:**
<1-2 sentences of user impact — filmmaker workflow terms, not architecture.>

**Environment:** macOS version, app version, model (for transcription issues).

<For ideas only, optional close: "Happy to share a focused patch if useful" — offer, never attach.>
